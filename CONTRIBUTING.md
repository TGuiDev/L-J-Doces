# Guia de Contribuição

Obrigado por contribuir com o **L&J Doces**.

## Fluxo recomendado
1. Crie uma issue descrevendo problema ou melhoria.
2. Crie uma branch a partir da `main`.
3. Desenvolva a alteração com commits claros.
4. Abra uma Pull Request usando o template.
5. Aguarde revisão e ajustes, se necessário.

## Padrões
- Referencie requisitos impactados (`RF`, `RN`, `RNF`) na PR.
- Mantenha escopo pequeno por PR.
- Atualize documentação quando houver mudança de regra, funcionalidade ou qualidade.

## Convenção de commits (sugestão)
- `feat:` nova funcionalidade
- `fix:` correção de bug
- `docs:` documentação
- `refactor:` refatoração sem mudança funcional
- `test:` testes
- `chore:` manutenção

## Checklist antes de abrir PR
- [ ] Código testado localmente
- [ ] Documentação atualizada
- [ ] Requisitos citados e rastreados
- [ ] Sem segredos/senhas no código
